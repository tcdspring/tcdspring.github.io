apt-get使用代理

Jun 4, 2019 16:13 · 29 words · 1 minute read

在使用apt-get update时如果遇到dl.google.com肯定会失败了，所以有时候还需要设置代理才行，如果使用下面方式设置：

export http_proxy=http://127.0.0.1:8123
sudo apt-get update

目前是不可行的，因为apt-get已经不再使用这种方式来指定代理了

方法1：

sudo apt-get -c ~/apt_proxy.conf update

# apt_proxy.conf 内容：

Acquire::http::proxy "http://127.0.0.1:8123/";
Acquire::ftp::proxy "ftp://127.0.0.1:8123/";
Acquire::https::proxy "https://127.0.0.1:8123/";

方法2：

sudo apt-get -o Acquire::http::proxy="http://127.0.0.1:8123/" update
