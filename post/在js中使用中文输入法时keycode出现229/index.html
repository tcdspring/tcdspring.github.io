在js中使用中文输入法时keyCode出现229

2018-05-10
Electron
约 694 字 预计阅读 2 分钟

文章目录

在最近开发的IM中，QA测出了一个BUG，使用中文输入法输入@时，不会弹出待选列表，但是英文输入法就可以，很奇怪，打开调试工具一看才知道原来在jquery的keydown事件中，获取到的keyCode的值为229，并非所期待的值。

PS: IM使用Electron框架开发，在输入框中输入@时，会弹出可以@的人的列表

代码是这样的：

// 监听输入框的键盘事件
$('#chat-input-area').keydown(function (e) {
    if (e.key == '@') {
        // 逻辑处理...
    }
});

网上搜索了好多方案，结果都不行，这个问题貌似无解啊，原因呢，好像是因为中文输入法还在处理，会给调用者先返回一个值，但是js中接收了值就不会再处理了，所以中文输入法下拿到的keyCode全部都是229，既然网上没有好的方案，那只能采用最笨的方法了。 每次输入记录下输入框中的值，再次输入时拿当前的值和上次记录的值做diff，然后拿到差异，也就是最后输入的数据了，但是这个diff算法的效率成了关键了，一个字符一个字符的比对，那效率肯定不高了，拿到当前焦点的索引，然后减1，就可以得到最后的输入字符，但是获取当前的索引还是比较复杂的，尤其是输入框里可能会有表情等特殊字符，所以一向比较懒的我选择了diff方案： 用到了这个开源库，diff速度很快： https://github.com/jhchen/fast-diff 下面是部分代码：

const diff = require('fast-diff');
let inputContent = '';

// 监听输入框的键盘事件
$('#chat-input-area').on('input', function (e) {
    let innerText = $('#chat-input-area')[0].innerText;
    let d = diff(inputContent, innerText);
    // console.log('diff result = ', d);
    inputContent = innerText;
    
    let isAt = false;
    if (!empty(d)) {
        for (let i = 0; i < d.length; i++) {
            if (d[i][0] == diff.INSERT && d[i][1].trimLeft(' ') == '@') {
                isAt = true;
                break;
            }
        }
    }

    if (!isAt) {
        return;
    }
    
    // 处理@
    // ...
});

经测试，3k+的一篇文章copy进去，然后按下Shift + 2，@弹窗基本上是秒开，根本感觉不到慢，效率还可以。
